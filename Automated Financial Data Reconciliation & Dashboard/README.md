# Automated Financial Data Reconciliation & Dashboard

**Organization:** Deloitte — Shared Services
**Role Context:** Data Analyst / Business Analyst
**Tech Stack:** Alteryx Designer · Power BI Desktop · SQL · Excel · Python (pandas)

---

## Business Problem

Deloitte's Shared Services team processes financial data for multiple business units, leading to:
- Manual reconciliation errors across 5+ disparate source systems
- Delayed month-end close reporting (typically 3–4 days of manual effort)
- Inconsistent data formats between SAP, Excel exports, and bank feeds
- No real-time visibility into exception items or outstanding reconciling differences

---

## Solution Overview

Built an end-to-end automated reconciliation pipeline that ingests, cleanses, matches, and reports on 100K+ monthly transactions across five source systems.

## Alteryx Workflow — ETL Pipeline

**File:** `alteryx/financial_reconciliation.yxmd`
Open this file directly in Alteryx Designer 2023.2+.

### Workflow Stages

```
[INPUT] 6 sources
    ↓
[DATA CLEANSING] Nulls · Duplicates · Whitespace · Format standardization
    ↓
[SELECT] Rename columns, set data types, drop unused fields
    ↓
[FILTER] Valid SAP only | Approved journals | Matched bank | AP exceptions | Budget flags
    ↓
[FORMULA] Calculated fields: quarter, aging bucket, variance %, PO match flag
    ↓
[JOIN] SAP ↔ Bank (reconciliation) | SAP ↔ AP (3-way match)
    ↓
[SUMMARIZE] By department, quarter, status, aging bucket
    ↓
[PYTHON TOOL] Match scoring, risk tiering, duplicate hash detection, KPI computation
    ↓
[OUTPUT] reconciled_transactions.csv | exception_report.csv | ap_aging.csv | budget_vs_actual.csv | SQLite DB
```

### Tools Used
- **Input Data Tool** — CSV/Excel ingestion
- **Data Cleansing Tool** — nulls, whitespace, duplicates
- **Select Tool** — column renaming and type casting
- **Filter Tool** — valid/invalid record separation
- **Formula Tool** — derived fields (quarter, aging, flags)
- **Join Tool** — SAP↔Bank and SAP↔AP matching
- **Summarize Tool** — aggregation by dimension
- **Python Tool** — advanced scoring and KPI logic
- **Output Data Tool** — CSV + SQLite output

## SQL Database

**File:** `sql/database_schema.sql`

Star schema with:
- **4 Dimension tables:** `dim_department`, `dim_account`, `dim_vendor`, `dim_customer`, `dim_calendar`
- **6 Fact tables:** SAP transactions, bank statements, journals, AP, AR, budget
- **1 Reconciliation result table** with match score and risk tier
- **1 Exception log table**
- **5 Views** pre-built for Power BI direct query

**Analytical Queries** (`reconciliation_queries.sql`):
1. Reconciliation status summary by period
2. Reconciliation rate by department
3. SAP vs bank gap analysis
4. 3-way PO match validation
5. AP aging (cash flow impact)
6. AR collections dashboard
7. Budget vs actual variance
8. Exception summary by type
9. Month-over-month trend
10. Daily reconciliation status
11. Top exception contributors (drill-through)
12. Data quality scorecard

## Power BI Dashboard

**File:** `powerbi/dax_measures.dax` (measures) + `powerbi/dashboard_layout.json` (layout spec)

### 6 Dashboard Pages

| Page | Purpose |
|------|---------|
| **Executive Summary** | C-suite KPI cards, reconciliation rate trend, dept breakdown |
| **SAP vs Bank Reconciliation** | Matrix detail, scatter variance, unmatched table |
| **AP & AR Dashboard** | Aging stacked bar, collections waterfall, overdue tables |
| **Budget vs Actual** | Column chart, variance waterfall, department heatmap matrix |
| **Exception Report** | Full exception table with drill-through, trend line |
| **Data Quality Scorecard** | Source system quality gauge and scorecard table |

### Key DAX Measures
- `Reconciliation Rate %` — Matched / Total × 100
- `Total Variance Amount` — Sum of ABS variance
- `Three Way Match Rate %` — PO + GRN + SAP matched invoices
- `DSO` — Days Sales Outstanding (AR / daily revenue)
- `Rolling 3M Avg Recon Rate` — DATESINPERIOD 90-day average
- `Budget Utilization %` — Actual Spend / Revised Budget
- `MTD / QTD / YTD Reconciled` — Time intelligence measures

### Slicers
- Date Range (between slicer)
- Department (dropdown)
- Match Status (tile)
- Risk Tier (tile)

### Bookmarks
- Reset All Filters
- Show High Risk Only
- Current Month View
- Exceptions Only
- Q1 2024 Full View

## Python Cleansing Script

**File:** `python/data_cleansing.py` (runs inside Alteryx Python Tool)

### Functions
| Function | Purpose |
|----------|---------|
| `standardize_amount()` | Strip `$`, `,`, convert to float |
| `standardize_date()` | Parse 5 date formats to ISO |
| `standardize_text()` | Trim whitespace |
| `normalize_status()` | Map status variants to canonical values |
| `detect_duplicates()` | Hash-based composite key deduplication |
| `calculate_match_score()` | 0–100 scoring across 5 quality dimensions |
| `assign_risk_tier()` | Low / Medium / High from score |
| `tag_exception_reason()` | Human-readable exception labels |
| `compute_kpis()` | Top-level KPI summary row |
| `enrich_ap_aging()` | Cash flow priority + payment risk flags |
